How to produce workshop proceedings for the NAACL 2013 handbook

This page is for conference organizers (main conference, workshop & demo chairs, *SEM chairs) who need to put together a schedule for inclusion in the NAACL handbook. The handbook duties have been folded into the responsibilities of the publications co-chairs this year. Alex Clemmer is heading up the construction of the printed handbook; in addition to this, NAACL will also be producing a digital version of the handbook that will be browsable on computers and smart phones. Both of these proceedings will be generated from the same order text files that have been used in the past.

Introduction

Workshop proceedings will be communicated to the NAACL 2013 handbook team in a canonical format. This document provides a (sparsely-illustrated) procedure to generate your workshop proceedings in this format.

Instructions

Our goal in this section is to develop instructions that allow workshop chairs to easily generate proceedings in the correct format, while simultaneously avoiding softconf or START, as they greatly complicate the process of building the handbook. Instructions fall roughly into (1) obtaining the data, (2) building an order file, and (3) formatting the order to fit the workbook format.

Obtaining the proceedings

Our first task is to obtain the raw proceedings data.
  • First, log on to softconf. Your screen should look roughly like it does in the screen capture to the right.
  • On the righthand side is a column marked "Conference Program". There is a hyperlink at the bottom of this column, with the text "ACLPUB". Click it.
  • We now generate the proceedings. Click on the "Generate" tab, and under "Generate Button", click "ALL".
  • Wait for the generation to finish (this may take awhile).
  • Click the download button.
  • We're done obtaining the data. You may close the browser.

Building the order file

Now we need to get the data into shape for transmission to the handbook team.
  • Uncompress the tarball you downloaded in the previous step. If you called it proceedings.tgz, then (assuming you are in the correct directory) you might run tar xzf proceedings.tgz.
  • Obtain the aclpub software. One way to do this is to download the CVS repository using the command cvs -d :pserver:anoncvs@ftp.clsp.jhu.edu:/aclpub checkout aclpub.
  • Next we must aclpub/make/Makefile bookchair proceedings Makefile. Subsequently:
    • You may need to set the environment variable export ACLPUB= pwd/aclpub for the following step.
    • In previous years, occasi FIXME
    • Open the instructions located at aclpub/doc/bookchairs/index.html.
    • If you followed our instructions in section \ref{obtain} for obtaining the data, then you may skip to the next step.
    • Otherwise, follow the steps up to (but not including) the step to execute make get-order.
  • Now we must attempt to make the order file.
    • In previous years, it has occasionally been a problem that the Perl module Text-PDF was not found.
    • This can be found at http://search.cpan.org/~mhosken/Text-PDF-0.29a/.
    • Download and unpack it.
    • Run ln -s Text-PDF-0.XX/lib/Text ., where XX is the version number.
  • This results in a file called order. Now the only task left is properly formatting the order file to reflect the workshop proceedings.

Formatting the order file

The order file currently contains pointers to all the papers in your workshop. Your job is now to produce your workshop proceedings. Bulletted below is a list of instructions about how to format your proceedings. In figure 2, we see a successful layout designed using these rules, from last year's conference. In figure 3, we see the \LaTeX output of this layout design.
  • The lines in the file are initially formatted like this: 5 \# Building a large annotated corpus of English: the penn treebank. These lines corresponds to papers.
  • You must now reorder them according to when they are presented, and supplement them with a time slot, e.g., 5 \# 09:10--09:35 Building a large annotated corpus of English: the penn treebank.
  • Other tools at your disposal:
    • The format * DAY will cause \LaTeX to render the header marking the start of a new day.
    • The format = HEADLINE will cause \LaTeX to render that line as a session headline.
    • The format + TIME DESCRIPTION will cause \LaTeX to render that line as an additional item.
  • Once you have done this, you can follow the rest of the directions for proceedings, and send the results to the relevant parties.
